#                     SyharikVDSBot
Удобное управление вашим личным VDS с помощью Telegram-бота
------------------------------------------------------------------------------------------------------------------------

# Что умеет бот?
Бот умеет узнавать статус VDS серверов (CPU RAM Disk в % нагрузки)

Удалённо выполнять любые команды от лица пользователя (/e ls)

Имеются быстрые команды по типу (/ns (netstat), /update (apt update), /reboot (reboot), /logs /var/logs/primer 50)

Так-же реализована функция установки и загрузки любого файла с сервера на сервер (/download , /upload)

При переходе установленных значений нагрузки CPU RAM хранилища и т.п. будет лететь варнинг.

# Настройка

scripts/start-syharikbot.sh - файл/скрипт для автоматического запуска бота в виртуальном окружении. Замените значения на свои

scripts/startpip-syharikbot.sh - файл/скрипт для автоматического запуска бота основном окружении. Это нужно для работы bash команд. Замените значения на свои

scritps/syharikbot.service - файл сервиса для автоматического запуска и дальнейшего хоста бота. Замените значения на свои. (Настроен запуск из основного окружения)

------------------------------------------------------------------------------------------------------------------------
config.json - файл для добавления своих VDS серверов. Заменить значения на свои. При необходимости добавить ещё несколько своих серверов.

Переменные:

"name": Название сервера для отображения в боте

"host": IP адресс удалённого сервера

"user": Имя пользователя для попытки подключения (SSH)

"auth_method": Метод авторизации. Есть 3 значения. "none" - отсутствует, "password" - При подключении бот будет просить пароль, "key" - использовние SSH ключа. Требуется дополнительная настройка!!!

Если был выбран метод "key", обязательно нужно добавить переменную "key_path": и в ней ввести путь до файла ключа.

Пример:

"key_path": "/path/to/key/id_rsa"

------------------------------------------------------------------------------------------------------------------------
main.py - основной файл бота. внутри так-же нужно заменить все значения на свои:

BOT_TOKEN = 'Ввести свой токен бота'

AUTHORIZED_USER_IDS = [Ввести ID пользователей-администраторов] (можно через запятую)

CONFIG_FILE = 'config.json' - при необходимости смените название.

MONITORING_ENABLED = True/False (включить мониторинг или нет)

CPU_THRESHOLD = 90.0 (процент заполненности для срабатывания варнинга)

RAM_THRESHOLD = 90.0 (процент заполненности для срабатывания варнинга)

DISK_THRESHOLD = 85.0 (процент заполненности для срабатывания варнинга)

MONITORING_INTERVAL = 60 (Интервал обновления данных в секундах)

------------------------------------------------------------------------------------------------------------------------

# Ручной
Перейдите в директорию с ботом:
cd /path/to/project

Создайте виртуальное окружение:
python -m venv .venv

Активируйте окружение:
source .venv/bin/activate

Установите зависимости:
pip install -r requirements.txt

Запустите бота:
python3 main.py

# Последующие запуски

Для вечного хоста данного бота можно настроить .service в etc/systemd/system
